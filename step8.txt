Этап 8. Apache + MariaDB + веб-приложение на SERVERЧто будет: на SERVER развернём веб-сервер Apache, базу MariaDB, создадим БД webdb, пользователя web, простую страницу index.php. Сайт доступен по http://192.168.60.2/ изнутри и через проброс http://<R1_WAN>:8080/ извне.Важно: в гайде модуля 2 предполагается, что у тебя есть Additional.iso с готовыми dump.sql и index.php. У тебя его нет, поэтому мы создадим простую тестовую страницу и базу с нуля — это полностью соответствует требованиям задания (создать БД, импортировать данные, проверить работу).1. SERVER — настраиваем MariaDBbashsystemctl enable --now mariadb
systemctl status mariadb --no-pager | head -5Должно быть active (running).Создаём БД, пользователя, таблицу и данные. Заходим в MariaDB:bashmysql -u rootВ консоли mysql выполняем по строкам:sqlCREATE DATABASE webdb;
CREATE USER 'web'@'localhost' IDENTIFIED BY 'P@ssw0rd';
GRANT ALL PRIVILEGES ON webdb.* TO 'web'@'localhost';
FLUSH PRIVILEGES;
USE webdb;
CREATE TABLE info (id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(50), role VARCHAR(50));
INSERT INTO info (name, role) VALUES ('Сервер предприятия', 'Web + DB + DNS');
INSERT INTO info (name, role) VALUES ('Отдел разработки', 'VLAN 20');
INSERT INTO info (name, role) VALUES ('Отдел дизайна', 'VLAN 30');
INSERT INTO info (name, role) VALUES ('Администрация', 'VLAN 10');
SELECT * FROM info;
EXIT;После SELECT * FROM info; должна вывестись таблица с 4 строками. Если всё ок — EXIT;.2. SERVER — создаём страницу index.phpbashnano /var/www/html/index.phpВставь:php<!DOCTYPE html>

Проверка изнутри сетиС SERVER (локально):bashcurl -s http://localhost/ | head -20Должно вернуть HTML с заголовком «Лабораторный стенд предприятия».С ADMINS:bashcurl -s http://192.168.60.2/ | head -20То же самое. Если хочешь — открой в графическом браузере на ADMINS (если установлен) и зайди на http://server.au-team.irpo/ — должна загрузиться красивая страничка с таблицей.5. Проверка извне через проброс портовНа R1 у нас проброс 8080 → 192.168.60.2:80. С R2 проверь:bashcurl -s http://10.0.137.158:8080/ | head -20(10.0.137.158 — это WAN-адрес R1, подставь свой если другой)Должна вернуться та же страница — это значит, проброс работает.Если что-то не работаетcurl localhost не работает на SERVER:bashjournalctl -xeu apache2 --no-pager | tail -20
